# Backlog

Задачи на основе отчёта тестирования (2026-03-03, 6 персон, 102 сообщения).
Общая оценка: **3.36 / 5.0**.

---

## P0 — Критические

### 1. Обрезка сообщений
**Частота**: 6/6 персон, 25+ случаев
**Проблема**: Сообщения обрываются на полуслове ("Хочешь точ", "Подойдут 2", "Сколько", "Пожал"). Пользователи тратят сообщения на "допиши", "обрезалось". Разрушает UX и доверие.
**Вероятная причина**: Лимит Telegram API (4096 символов) или стриминг отправляет неполный ответ.
**Решение**: Дождаться полного ответа перед отправкой; если ответ длиннее лимита — разбить на части автоматически.

### 2. Уведомление о памяти вместо ответа
**Частота**: 3/6 персон (Анна — 2 раза, Лиза)
**Проблема**: Бот отправляет ТОЛЬКО блок "📝 Обновил память..." без ответа на вопрос пользователя. Пользователь не получает то, что просил.
**Решение**: Уведомление о памяти никогда не должно отправляться без основного ответа. Если ответ + уведомление не помещаются — убирать уведомление, не ответ.

## P1 — Высокие

### 3. Избыточные уведомления о памяти
**Частота**: 6/6 персон, 3-5 раз за сессию
**Проблема**: Блок "📝 Я обновил то, что помню о тебе..." с подробным diff'ом появляется слишком часто, занимает больше текста чем сам ответ, раздражает особенно лаконичных пользователей.
**Решение**: Варианты (от лучшего к худшему):
- Молча обновлять (без уведомления)
- Короткое "Запомнил" / "Запомнила 💅" без diff'а
- Показывать diff только по запросу (/memory)

### 4. Молчание / таймауты
**Частота**: 3/6 персон (Маша — 2 раза, Анна — 3 раза)
**Проблема**: Бот не отвечает вообще, пользователь вынужден "толкать" повторным сообщением. Создаёт ощущение ненадёжности.
**Решение**: Расследовать причину (таймаут inference, потеря сообщения, ошибка в pipeline). Добавить fallback-ответ при таймауте.

### 5. Игнорирование явных предпочтений пользователя
**Частота**: 2/6 персон (Дмитрий, Виктор)
**Проблема**: Пользователь просит "без уведомлений о памяти" — бот продолжает их отправлять. Исправляется только после повторного замечания.
**Решение**: Предпочтения вроде "без уведомлений" должны применяться немедленно и надёжно. Проверить, сохраняются ли такие указания в промпте/снапшоте и передаются ли в inference.

## P2 — Средние

### 6. Плохой первый ответ
**Частота**: 2/6 персон (Маша, Олег)
**Проблема**: На "что ты умеешь?" бот ответил одним словом "Привет". На "Ну и что тут делать" — "Что именно". Убийство мотивации для новых пользователей.
**Решение**: Добавить в системный промпт инструкцию: на вопросы о возможностях всегда давать конкретные примеры пользы, привязанные к интересам пользователя из онбординга. Никогда не отвечать одним словом на первое сообщение.

### 7. Не реагирует на прощание
**Частота**: 2/6 персон (Маша, Олег)
**Проблема**: На "пока!", "спасибо, до свидания" бот повторяет информацию из памяти или даёт непрошеные советы вместо прощания.
**Решение**: Добавить в промпт инструкцию распознавать прощание ("пока", "до свидания", "спасибо, всё") и отвечать уместно — коротко попрощаться.

### 8. Повтор одобренного текста
**Частота**: 1/6 персон (Олег)
**Проблема**: Пользователь сказал "годится", бот повторил весь текст жалобы заново. Раздражает.
**Решение**: Добавить в промпт: если пользователь одобрил текст — не повторять его, просто подтвердить.

### 9. Додумывает бизнес-решения за пользователя
**Частота**: 1/6 персон (Виктор)
**Проблема**: Бот предложил скидку 10% в ответе клиенту без запроса. Скидки, компенсации, сроки — решения владельца бизнеса.
**Решение**: Добавить в промпт: не предлагать финансовые обязательства (скидки, возвраты, компенсации) без явного указания пользователя. Вместо этого — оставить плейсхолдер или спросить.

### 10. Непоследовательность ты/вы
**Частота**: 2/6 персон (Олег, Виктор)
**Проблема**: Бот произвольно переключается между "ты" и "вы", нарушая тональность.
**Решение**: Определять форму обращения по стилю пользователя с первого сообщения и фиксировать в снапшоте. Если неоднозначно — использовать "ты" для дружелюбного/игривого тона, "вы" для делового/лаконичного.

## P3 — Улучшения

### 11. Длинные списки разбивать проактивно
**Проблема**: Контент-план на 7 дней потребовал 4 сообщения из-за обрезки.
**Решение**: Если ответ будет длинным — предупредить "Пришлю частями" и разбить самостоятельно.

### 12. Плейсхолдеры для несуществующих артефактов
**Частота**: 1/6 персон (Дмитрий)
**Проблема**: Бот вставил "[ссылка на чеклист]" в текст, хотя чеклиста нет.
**Решение**: Не вставлять плейсхолдеры для артефактов, о которых пользователь не упоминал.

### 13. Избыточные уточняющие вопросы
**Частота**: 1/6 персон (Маша)
**Проблема**: 3 уточняющих вопроса подряд вместо цитат. Затягивает диалог.
**Решение**: Лучше дать примерный ответ и предложить уточнить, чем задавать серию вопросов.
