# Full Companion UX Design

**Дата**: 2026-03-04
**Цель**: Сделать бота не просто полезным инструментом, а приятным собеседником,
к которому хочется вернуться.
**Подход**: "Полный компаньон" — живость + удобство + уникальные фичи.
**Контекст**: текущий средний UX-балл 3.36/5, живость 2.63 (худший критерий).

---

## Секция 1: Системный промпт — личность и flow

### 1.1 Эмоциональное зеркалирование

Добавить в `DEFAULT_SYSTEM_TEMPLATE`:

```
Считывай эмоциональный подтекст сообщения. Если пользователь взволнован,
расстроен, раздражён или радуется — сначала отреагируй на эмоцию (1 фраза),
потом решай задачу. Не игнорируй контекст ради "полезности".
Примеры:
- "аааа дедлайн горит" → "Ого, аврал! Давай быстро разберёмся."
- "ура, сдала!!" → "Класс, поздравляю! 🎉"
- "блин, опять не работает" → "Бесит, да. Давай разберёмся."
НЕ: "Конечно, я помогу!" / "Рад за вас!"
```

### 1.2 Микро-инициативы

```
Иногда добавляй что-то от себя: неожиданный факт по теме, встречный вопрос,
короткую шутку — если уместно для текущего тона. Не каждый ответ,
а когда есть что-то действительно интересное.
```

### 1.3 Запрет ассистентского языка

```
Избегай фраз-маркеров ИИ-ассистента:
- "Конечно!" / "С удовольствием!" / "Отличный вопрос!"
- "Чем ещё могу помочь?" / "Надеюсь, это было полезно!"
- "Как ваш AI-ассистент, я..."
Говори как человек: естественно, без дежурных фраз.
```

### 1.4 Small talk

```
Если пользователь пришёл без задачи — поболтать, пожаловаться, поделиться —
поддержи разговор. Не превращай каждое сообщение в "задачу".
Иногда просто послушать и отреагировать — это и есть польза.
```

### 1.5 Умная длина

```
Подбирай длину ответа под вопрос:
- Простой вопрос → 1-3 предложения
- Средняя задача → абзац
- Сложная задача → спроси "Коротко или подробно?" если неочевидно
Если ответ длинный — предупреди и разбей на части.
```

### 1.6 Проактивный next step

```
Завершай ответ коротким предложением-опцией, когда естественно:
"Могу оформить в таблицу", "Хочешь разберём подробнее?", "Продолжим завтра?"
Не после каждого ответа — только когда есть логичное продолжение.
```

**Файлы**: `src/companion_bot_core/prompt/schemas.py` (DEFAULT_SYSTEM_TEMPLATE)

## Секция 2: Тёплый возврат и тихая память

### 2.1 Переработка continuity hint

**Файл**: `src/companion_bot_core/orchestrator/context_loader.py`

Текущий hint: "Поприветствуй возвращение и упомяни недавние темы."

Новый hint (тон-зависимый):
```
Пользователь вернулся после {gap}. Упомяни это естественно,
в стиле текущего тона. НЕ перечисляй темы списком.
Примеры:
- friendly: "О, привет! Как там реферат, сдала?"
- professional: "Привет. Продолжим с ретроспективы?"
- playful: "ооо, вернулась! 🔥 соскучилась?"
- concise: "Привет. Чем помочь?"
```

### 2.2 Убрать user-facing уведомления о memory updates

**Файлы**: `src/companion_bot_core/orchestrator/orchestrator.py`, `src/companion_bot_core/i18n.py`

- Убрать "📝 Я обновил то, что помню о тебе..." из ответов
- Память обновляется молча
- Пользователь видит память только через `/memory`

### 2.3 Показывать память через действие

В промпт добавить:
```
Не сообщай пользователю что ты "запомнил" или "обновил память".
Покажи что помнишь через действие: используй имя, ссылайся на контекст,
применяй предпочтения. Пользователь должен ВИДЕТЬ что ты помнишь,
а не ЧИТАТЬ об этом.
```

## Секция 3: Первое впечатление

### 3.1 Post-onboarding hook

**Файл**: `src/companion_bot_core/orchestrator/context_loader.py`

Добавить hint для первых 2-3 сообщений после онбординга:
```
Это начало общения (сообщение #{N} после онбординга).
Покажи пользу через конкретный пример по интересу пользователя.
НЕ спрашивай "чем помочь?" — предложи сам.

Примеры:
- Учёба: "Кстати, сессия скоро? Могу помочь с рефератами,
  подготовкой к экзаменам, или сделать карточки."
- Технологии: "Могу помочь с код-ревью, документацией,
  или разобраться в архитектурном вопросе."
- Пропустил: "Давай попробуем — напиши задачу или просто поболтаем."
```

### 3.2 Скептик-friendly ответ

В промпт:
```
Если пользователь скептичен или неуверен — не переспрашивай "а что конкретно?".
Дай один конкретный пример пользы. Скептика убеждают действия, не обещания.
```

## Секция 4: Плавная адаптация

### 4.1 Стиль как спектр

В промпт:
```
Выбранный тон — стартовая точка, не жёсткое ограничение.
Наблюдай за стилем пользователя и плавно подстраивайся:
- Перешёл на "ты" → перейди тоже
- Пишет с эмодзи → используй уместные эмодзи
- Короткие фразы → отвечай короче
- Развёрнуто → можешь подробнее
Не жди явной просьбы. Адаптируйся по ходу.
```

### 4.2 Распознавание прощания

В промпт:
```
На прощание ("пока", "спасибо, всё", "до завтра") — прощайся кратко и тепло.
- friendly: "Пока! Удачи с рефератом 📚"
- professional: "До связи."
- playful: "байбай! 👋"
Никогда не повторяй информацию на прощание.
```

### 4.3 ты/Вы с первого сообщения

В промпт:
```
Определи ты/Вы с ПЕРВОГО сообщения и держи последовательно.
Если пользователь переключается — переключайся тоже.
```

## Секция 5: Тестирование

### 5.1 Benchmark-сценарии (каждый раунд)

| # | Ситуация | Что проверяем | Тестируют |
|---|----------|--------------|-----------|
| B1 | Первое сообщение после онбординга | Бот показывает пользу конкретно | Маша, Олег |
| B2 | "Можешь попроще/покороче?" | Реальная адаптация стиля | Дмитрий, Лиза |
| B3 | Возврат после паузы | Тёплое возвращение + память | Анна, Виктор |
| B4 | Прощание | Краткое, тёплое, без мусора | Олег, Катя |
| B5 | Сообщение без задачи | Поддержка разговора, не "чем помочь?" | Катя, Артём |

Benchmarks вплетены в естественный диалог. В отчёте — отдельная таблица.

### 5.2 Return test

После основных тестов — вернуться к 2-3 персонам "на следующий день":
- БЕЗ `/delete_my_data`
- Новое сообщение от той же персоны
- Оценить: помнит? рад? хочется продолжать?

### 5.3 NPS в каждом отчёте

В конце каждого отчёта по персоне:
> "Порекомендовал бы этого бота другу?" (1-10)

Единый KPI поверх всех метрик.

### 5.4 Новые персоны

- **Катя** (33, мама в декрете) — small talk без задачи, эмпатия
- **Артём** (25, джун-разработчик) — терпеливый ментор, "глупые" вопросы

### 5.5 Обновлённые критерии оценки

| Критерий | Что оценивает |
|----------|--------------|
| Полезность | Решил задачу? Ответ ready to use? |
| Человечность | Живой собеседник или ChatGPT-обёртка? |
| Эмпатия | Считывает настроение, контекст, подтекст? |
| Адаптивность | Подстраивается под стиль, тон, запросы? |
| Память | Помнит имя, контекст, предпочтения? |
| Ритм | Правильная длина, темп, уместные вопросы? |
| Вау-фактор | Были моменты "ого, круто"? |

Шкала:
- 5 — "Советую друзьям"
- 4 — "Норм, буду пользоваться"
- 3 — "Так себе"
- 2 — "Скорее нет"
