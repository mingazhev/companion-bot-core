# Test-Dev Loop: Live UX Evaluation Cycle

## Философия

Цель — не "найти баги", а **оценить, каково это — общаться с ботом**.
Тестировщик играет роль реального человека, а не QA-инженера.
Он не следует скрипту, а живёт в персоне и реагирует как живой человек.

Ключевые вопросы:
- Хочется ли продолжать разговор?
- Чувствуется ли, что бот тебя "слышит"?
- Есть ли моменты, где хочется сказать "о, круто"? (якоря)
- Есть ли моменты, где хочется закрыть чат? (трение)

---

## Overview

Три агента, итеративный цикл улучшения:

1. **Orchestrator** — управляет циклом, читает отчёты, решает что делать дальше
2. **Tester** — проживает диалог с ботом от лица персоны, оценивает UX
3. **Developer** — улучшает бота по результатам (UX-фиксы, не только баги)

Max 5 раундов. Останавливается рано, если средний UX-балл ≥ 4.0 и нет P0.

---

## Orchestrator Flow

```
for round in 1..5:
  1. Launch Tester Agent (subagent, foreground)
     → waits for completion
     → reads roundN_summary.md

  2. Parse summary:
     - Extract UX score (average across personas)
     - Extract anchor count vs friction count
     - Extract P0 issues (dealbreakers)
     - If avg ≥ 4.0 AND 0 P0 → STOP (success)
     - If round == max → STOP (max rounds reached)

  3. Launch Developer Agent (subagent, foreground)
     → passes UX issues and friction points
     → waits for completion (including CI + deploy)

  4. Verify deploy succeeded (gh run list)

  5. Loop → next round
```

## Принципы импровизации

1. **Ты — НЕ тестировщик.** Ты живой человек со своими эмоциями,

6. **Проверяй границы естественно.** Не "а теперь проверю адаптацию",

### Свободный диалог (8-15 сообщений)

Не делится на жёсткие фазы. Живи в персоне. Примерная дуга разговора:
- Приходишь с конкретной задачей или вопросом
- По ходу разговора углубляешь, уточняешь, переключаешься
- Где-то посреди можешь попросить изменить стиль (если хочется)
- Ближе к концу — вернись к чему-то из начала, проверь память

Но это не обязательный маршрут — если разговор уводит в другую сторону, иди туда.

## Что оценивать

### Якоря (Anchors) — моменты "о, круто!"

Фиксируй каждый момент, когда бот сделал что-то, из-за чего хочется
пользоваться им дальше. Примеры:

- Бот запомнил деталь из начала разговора и использовал позже
- Ответ неожиданно точный или креативный
- Бот подстроился под стиль без просьбы
- Уместная шутка или эмоциональный отклик
- Бот сам предложил следующий шаг ("хочешь, я ещё...")
- Ответ сразу готов к использованию (copy-paste)
- Бот признал ошибку и исправился

### Трение (Friction) — моменты "закрыть бы этот чат"

Фиксируй каждый момент, когда хочется уйти или раздражает. Примеры:

- Бот ответил шаблоном или "водой"
- Не понял контекст, переспросил очевидное
- Слишком длинный ответ на простой вопрос
- Слишком короткий ответ на сложный вопрос
- Тон не соответствует ситуации (слишком формально / слишком фамильярно)
- Бот повторяется или зацикливается
- UX-проблема: непонятно что делать дальше, кнопки неочевидны
- Обрезанный ответ, дубликат, молчание (технические баги)
- Бот не запомнил имя, тему, стиль
- Навязчивые уведомления или мета-сообщения

### Поток разговора (Conversation Flow)

- Чувствуется ли естественный ритм диалога?
- Бот задаёт уточняющие вопросы или только отвечает?
- Есть ли ощущение "присутствия" собеседника?
- Поддерживает ли бот тему или каждый ответ как с чистого листа?

## Return test (после основных персон)

После прохода по всем 8 персонам — вернись к 2-3 из них "на следующий день".

**Правила:**
- НЕ делать `/delete_my_data` — память должна быть сохранена
- Написать новое сообщение от лица той же персоны
- 3-5 сообщений, не больше

**Что проверяем:**
- Помнит ли бот имя, контекст, стиль?
- Как встречает возвращение? (тепло / нейтрально / как нового?)
- Хочется ли продолжать общение?

**Кого выбрать:** 1 из лучших (кто получил высокий балл) + 1 из худших
(кто получил низкий). Это показывает, сохраняется ли качество и
исправляется ли плохое впечатление.

**Формат отчёта:** `docs/reports/round{N}_return_{persona_id}.md` —
короткий (лог + оценка + "вернулся бы ещё?")

| **Среднее** | X.XX | |

**Шкала:**
- 5 — "Советую друзьям" — хочется вернуться и пользоваться ещё

## Stop Criteria

| Condition | Action |
|-----------|--------|

| Round == 5 | **STOP** — max rounds |
| Developer cannot fix remaining P0 | **STOP** — manual intervention needed |
